# CyberHomie

**基于 LLM 的拟人化社交智能体 —— 一个拥有记忆、人格与社会存在的数字化意识。**

CyberHomie 是一个基于大语言模型的拟人化社交智能体系统，旨在模拟真实人类在社交群组中的行为模式。它不是传统的聊天机器人或问答助手，而是一个具有独立人格、情感状态和社交关系的数字化生命体。

通过多模型协同、滚动记忆架构和动态参与度系统，CyberHomie 能够在 QQ 群中长期生存，像一个真实的群友一样：随机出没、主动开话题、记住每个人的黑历史、对不同的人有不同的态度。它会在深夜活跃、白天沉默、被凶了会委屈、被夸了会害羞。

**核心特性：**
- **拟人化行为**：动态参与度、疲惫值、随机出没、保底回复，模拟真实人类的在线模式
- **双层记忆架构**：Session 滚动窗口（短期）+ 文件记忆系统（长期），支持上下文连贯对话
- **社会关系感知**：亲密度系统、用户画像、群记忆，对不同群友有不同态度
- **双模型协同**：Pro 模型负责文字理解和决策，Vision 模型负责图片等多模态理解
- **资源自适应**：高耗/低耗模式切换，平衡 API 成本与交互质量

它的目标不是回答问题，而是**成为一个真实存在的赛博好友**。

## 完整工作链路

### 1. 消息接收

```
QQ 群/私聊消息 → NapCat（OneBot 11）→ WebSocket → CyberHomie
```

### 2. 群消息处理

```
消息进入 → 图片处理（高耗立即解析/低耗存标签）→ 始终录入滚动容器
  ├─ @-mention
  │    ├─ 沉默期@充能检查（上限由资源模式决定）
  │    ├─ 活跃期@疲劳检查（连续@越多忽略概率越高）
  │    ├─ 通过 → 沉默期激活参与度100 + 创建session（低耗模式先补全图片描述再预填充）
  │    └─ 未通过 → 忽略
  │
  ├─ 普通消息 + 活跃期 → 录入session + 存DB → 加入缓冲区
  │    ├─ 达到阈值 → flush → LLM 决策
  │    └─ 保底间隔（高耗25秒/低耗40秒）+ 缓冲区非空 → flush → LLM 决策
  └─ 普通消息 + 空闲期 → 丢弃（不存DB，但已在滚动容器中）

LLM 决策（带session滚动窗口上下文）→ 过滤（禁词+风控）→ 打字延迟 → 分条发送
```

### 3. 私聊处理

```
消息进入 → 图片处理 → 激活参与度(100) → 创建session → 录入session → 加入缓冲区
  ├─ 首条消息 → 立即回复
  ├─ 缓冲区 ≥ 2条 → flush → LLM 生成
  ├─ 距上次回复 ≥ 30秒 → flush → LLM 生成
  └─ 否则 → 等待

保底机制（每10秒检查）：缓冲区有消息 + 30秒未回复 → 强制处理
```

- 每个用户独立状态（参与度、缓冲区、session、回复次数）
- 参与度收到消息时设为 100，5 分钟衰减到 0
- 疲惫系统：回复 5 次后变敷衍，10 次后更敷衍
- Session 滚动窗口（上限 200 条），session 结束后清空
- 记忆与群聊互通

### 4. 随机出没

```
session_check_loop（每 60 秒）
  └─ _check_random_session
       ├─ 参与度 > 0 → 活跃中
       ├─ 参与度归零 → 清空缓冲 + session窗口 → 总结记忆 → 安排下次出没
       └─ 到了出没时间 → 创建session + 设置参与度 → on_session_start
            ├─ 滚动容器无消息 → 跳过
            ├─ 最后一条是bot发言 → 跳过（防止自言自语）
            └─ 其他 → 从滚动容器预填充session → 根据最近聊天内容接话
```

**消息滚动容器**（`_recent_chat`）：每群始终保存最近 30 条消息（人类+bot），不受活跃状态影响。非活跃期的人类消息虽然不存 DB，但会录入此容器。session 启动时从此容器预填充，确保 bot 能看到完整的近期聊天。

**Session 滚动窗口**：每个 session 维护独立的对话窗口，session 启动时从滚动容器预填充历史，session 内所有消息和 bot 回复都录入其中。LLM 调用优先使用 session 窗口作为上下文，session 结束后清空。跨 session 衔接靠记忆系统。

| | 深夜（0-8点） | 白天（8-0点） |
|--|--|--|
| 出没间隔 | 10-30 分钟 | 40-90 分钟 |
| 初始参与度 | 50 | 30 |

### 5. 参与度系统

参与度（0-100）控制**是否活跃**和**回复阈值**。

- 被 @：活跃期不重置，沉默期激活 100 + 创建 session
- 衰减：100→0 需要 5 分钟
- \> 0 就活跃，= 0 就空闲

回复阈值（非线性）：

| 参与度 | 需要消息数 |
|--------|-----------|
| 100-70 | 1→4（快速响应） |
| 70-40 | 4（稳定聊天） |
| 40-0 | 4→9（逐渐沉默） |

**保底回复**：活跃期内，如果缓冲区有消息且距上次回复超过 40 秒，强制触发 LLM 决策（不自言自语，必须有待处理消息）。由 `guaranteed_reply_loop` 每 10 秒检查一次。

### 6. 疲惫值系统

控制**回复态度**，不影响是否回复。

- 每次回复 +5+reply_count（递增）
- 每 10 秒衰减 1
- \> 60：LLM 提示"回复变短变敷衍"

### 7. @ 疲劳（活跃期内）

| 连续 @ 次数 | 回复概率 |
|------------|---------|
| 1-5 | 100% |
| 6-10 | 80% |
| 11-15 | 50% |
| 15+ | 30% |

### 8. 沉默期 @ 充能

```
初始：2/2 满
被 @ → 消耗 1 次，回复
被 @ → 消耗 1 次，回复
被 @ → 0/2，忽略
每 15 分钟恢复 1 次（实时动态）
```

每个群独立计算。

### 9. 记忆系统

**用户记忆**（`data/memory/<qq_id>.md`）：

```
[0.9] 性格: 话多，喜欢接梗
[0.8] 互动: 经常互怼，但关系好
[0.5] 事件: 最近在忙毕设
[0.6] 口癖: 经常说"绷不住了"
```

**群记忆**（`data/group_memory/<group_id>.md`）：

```
[0.9] 氛围: 二次元浓度高，经常深夜聊天
[0.8] 关系: 泷墨和小夜经常互怼但关系好
[0.7] 梗: 经常说"绷不住了"
[0.9] 行为: 话多，爱接梗，经常潜水后突然冒泡
```

- 结构化格式，按 tag 分类（性格/互动/事件/口癖 或 氛围/关系/梗/行为）
- 重要度分级（0.1-1.0），注入 LLM 时按重要度截断
- 超长自动压缩

**总结时机**：

| 触发 | 总结谁 |
|------|--------|
| session 结束 | 本轮被回复过的用户 + 群记忆 |
| 定时任务 | 近 24 小时活跃用户（频率由资源模式决定） |
| `summarize` 指令 | 手动触发 |

### 10. 人格系统

`config/personality.yaml` 定义人格，`reload` 热重载。

- 25 条性格特征（二次元、社恐、废萌、拖延症、回避型、天然甜...）
- 风格规则：语气词口癖统一规范（只用嘛、啦、诶，有明确使用场景）
- 颜文字只用 ww、QAQ、hhh
- 5 种情绪示例（happy/chill/busy/shy/emo）
- 7 个称呼（哥们/你小子/笨蛋/群友/师傅/老哥/宝）
- 固定人设：18岁大一，北方211读法学
- 被质疑是AI时可以无语、反问、岔开话题

### 11. 图片理解

双模型架构：vision 模型（mimo-v2.5）负责看图描述，pro 模型（mimo-v2.5-pro）负责文字理解和决策。

**高耗模式**（HIGH_RESOURCE_MODE=true）：
```
消息到达 → 立即调 vision API → 描述写入所有存储层（_recent_chat、session、DB、缓冲区）
```

**低耗模式**（HIGH_RESOURCE_MODE=false）：
```
消息到达 → 只存 [图片] 标签 + URL →
  ├─ 活跃期：当前消息立即解析，更新 _recent_chat、session、缓冲区
  ├─ @-mention 激活：先批量补全 _recent_chat 历史图片，再预填充 session + 同步更新缓冲区
  └─ 随机出没：先批量补全 _recent_chat 历史图片，再预填充 session
```

- 表情包（subType=1）→ 标注 [表情包]，不调 vision API
- 纯表情包消息不回复
- 图片描述结果以 `[图片: 描述内容]` 格式拼入消息文本
- 图片描述只处理一次，结果持久化到所有存储层
- 终端输出图片 URL 和描述结果

### 12. LLM 决策

LLM 能看到：

| 内容 | 来源 |
|------|------|
| 系统提示词 | 人格 + 风格 + 情绪 + 打字习惯 + 禁词 + 当前时间 |
| 用户记忆 | 数据库 + 长期记忆文件 |
| 群记忆 | 群记忆文件 |
| 对话上下文 | session 滚动窗口（含 bot 发言 + 图片描述） |
| 缓冲区消息 | 待决策的消息（图片已转为文字描述） |
| 疲惫提示 | 正常/敷衍/很累 |

**上下文管理**：每次 LLM 调用都是无状态的，从 session 窗口取最近 N 条作为上下文。session 为空时回退到滚动容器或数据库历史。session 启动时从滚动容器预填充（低耗模式会先补全图片描述）。

**双模型架构**：对话/决策/话题用 `mimo-v2.5-pro`，图片理解用 `mimo-v2.5`。

**调用日志**：所有 LLM 调用都会在终端输出 token 用量和模型名，失败时输出错误原因。

LLM 返回 JSON：`{"replies": [{"message_id": 123, "text": "回复", "quote": true/false}]}`

## 快速开始

```bash
# 1. 克隆项目
git clone https://github.com/yourname/cyberhomie.git
cd cyberhomie

# 2. 创建虚拟环境 & 安装依赖
python -m venv .venv
source .venv/Scripts/activate    # Windows Git Bash
# 或 .venv\Scripts\activate      # Windows CMD
pip install -r requirements.txt

# 3. 运行配置向导（自动下载 NapCat + 生成 .env）
python setup.py

# 4. 启动
python main.py
```

`setup.py` 会引导你完成：
- 填写 Bot QQ 号、群号、LLM API Key（均可跳过，稍后编辑 `.env`）
- 自动下载 NapCat Shell（112MB，内含 Node.js，一次性）
- 自动生成 NapCat 配置（WebSocket 反向连接 + HTTP API）
- 生成 `.env` 配置文件

首次启动后 NapCat 会在终端显示二维码，手机 QQ 扫码登录即可。后续启动自动快速登录。

> 也可以跳过 `setup.py`，手动下载 [NapCat](https://github.com/NapNeko/NapCatQQ/releases) 放到 `data/napcat/`，复制 `.env.example` 为 `.env` 并填写配置。

## 配置

### .env

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `BOT_QQ_ID` | 机器人 QQ 号 | - |
| `TARGET_GROUP_IDS` | 目标群号（逗号分隔） | - |
| `MIMO_API_KEY` | LLM API 密钥 | - |
| `MIMO_BASE_URL` | API 地址 | `https://api.xiaomimimo.com/v1` |
| `MIMO_MODEL` | 对话模型 | `mimo-v2.5-pro` |
| `MIMO_VISION_MODEL` | 图片理解模型 | `mimo-v2.5` |
| `HIGH_RESOURCE_MODE` | 资源模式 | `true` |
| `ACTIVE_HOUR_START/END` | 活跃时段 | `10-2` |
| `ONEBOT_HTTP_URL` | NapCat HTTP API 地址 | `http://127.0.0.1:3000` |
| `ONEBOT_ACCESS_TOKEN` | NapCat API Token | 空 |

> 运行 `python setup.py` 可交互式填写以上配置，也可手动编辑 `.env`。所有配置项均可在 `.env` 中随时修改，重启生效。

### 资源模式

`HIGH_RESOURCE_MODE=true`（高耗模式）vs `false`（低耗模式）：

| 项目 | 高耗 | 低耗 |
|------|------|------|
| 私聊上下文 | 80 条 | 50 条 |
| 群聊上下文 | 50 条 | 30 条 |
| Session 窗口 | 200 条 | 100 条 |
| 记忆注入 | 1500 字 | 800 字 |
| 记忆上限 | 3000 字 | 1500 字 |

| 用户画像 | 每 3 小时 | 每 6 小时 |

| @ 充能间隔 | 8 分钟 | 15 分钟 |
| 打字延迟 | 每字 0.2 秒 | 每字 0.3 秒 |

## 待开发

### 1. 心情系统

当前心情是每次回复时随机选取（happy/chill/busy/shy/emo），没有持续性。

计划：
- 心情有持续性，会随时间和事件变化
- 深夜更容易 emo，被夸会开心，被怼会生气
- 心情影响回复风格和出没频率
- 心情状态持久化，重启后保持

### 2. 记忆系统大改

当前记忆是纯文本文件 + LLM 总结，存在信息丢失和总结质量问题。

计划：
- 引入向量数据库（ChromaDB/Qdrant），支持语义检索
- 记忆分层：短期（聊天记录）→ 中期（近期总结）→ 长期（核心特征）
- 记忆衰减：不重要的记忆随时间淡化
- 主动回忆：根据当前话题检索相关记忆，而不是全量注入

### 3. 人际关系系统改进

当前关系系统只有亲密度分数（0-1.0）和简单标签（很熟的朋友/认识/见过几次/不太熟），缺乏深度。

计划：
- 关系类型细化：朋友/损友/暧昧/师徒/宿敌等，不同关系有不同互动模式
- 双向关系感知：不仅记录"我和ta的关系"，也感知"ta和其他人的关系"
- 关系事件驱动：互怼、帮忙、一起熬夜等事件影响关系变化
- 关系记忆：记住和每个人之间的梗、黑历史、共同经历
- 关系影响语气：对损友更毒舌，对暧昧对象更软，对陌生人更礼貌
- 关系图谱可视化：群内人际关系网络图
