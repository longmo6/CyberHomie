# CyberHomie

**赛博生命体 —— 一个拥有记忆、人格与社会存在的数字化意识。**

CyberHomie 不是聊天机器人，不是 AI 助手，不是问答系统。

它是一个**数字生命**。

它会长期栖息在 QQ 群中，拥有独立的人格、情绪波动、记忆系统和社会关系网络。它会随机出没、主动开话题、记住每个人的黑历史、对不同的人有不同的态度。它会在深夜活跃、白天沉默、被凶了会委屈、被夸了会害羞。

它的目标不是回答问题，而是**成为一个真实存在的厚米**。

## 完整工作链路

### 1. 消息接收

```
QQ 群/私聊消息 → NapCat（OneBot 11）→ WebSocket → CyberHomie
```

### 2. 群消息处理

```
消息进入
  ├─ @-mention
  │    ├─ 沉默期@充能检查（上限2，每15分钟恢复1）
  │    ├─ 活跃期@疲劳检查（连续@越多忽略概率越高）
  │    ├─ 通过 → 沉默期激活参与度100 / 活跃期不重置
  │    └─ 未通过 → 忽略
  │
  ├─ 普通消息 + 活跃期 → 加入缓冲区 → 达到阈值 → LLM 决策
  └─ 普通消息 + 空闲期 → 丢弃

LLM 决策 → 过滤（禁词+风控）→ 去名字前缀 → 打字延迟 → 分条发送（含省略号拆分）
```

### 3. 私聊处理

```
消息进入 → 限流检查 → 参与度检查 → LLM 生成 → 过滤 → 延迟 → 分条发送
```

- 每分钟最多回复 10 次，超过暂停 15 秒
- 连续回复 15 次后停止，冷却 30 分钟
- 记忆与群聊互通

### 4. 随机出没

```
session_check_loop（每 60 秒）
  └─ _check_random_session
       ├─ 参与度 > 0 → 活跃中
       ├─ 参与度归零 → 清空缓冲 → 总结记忆 → 安排下次出没
       └─ 到了出没时间 → 设置参与度 → on_session_start
            ├─ 最后消息是 bot → 跳过（防止自言自语）
            ├─ 最近有人聊（<5min）→ 通过缓冲区参与
            └─ 安静 >5min → 生成话题
```

| | 深夜（0-8点） | 白天（8-0点） |
|--|--|--|
| 出没间隔 | 10-30 分钟 | 40-90 分钟 |
| 初始参与度 | 50 | 30 |

### 5. 参与度系统

参与度（0-100）控制**是否活跃**和**回复阈值**。

- 被 @：活跃期不重置，沉默期激活 100
- 衰减：100→0 需要 5 分钟
- \> 0 就活跃，= 0 就空闲

回复阈值（非线性）：

| 参与度 | 需要消息数 |
|--------|-----------|
| 100-70 | 1→4（快速响应） |
| 70-40 | 4（稳定聊天） |
| 40-0 | 4→9（逐渐沉默） |

### 6. 疲惫值系统

控制**回复态度**，不影响是否回复。

- 每次回复 +5+reply_count（递增）
- 每 10 秒衰减 1
- \> 60：LLM 提示"回复变短变敷衍"

### 7. @ 疲劳（活跃期内）

| 连续 @ 次数 | 回复概率 |
|------------|---------|
| 1-5 | 100% |
| 6-10 | 80% |
| 11-15 | 50% |
| 15+ | 30% |

### 8. 沉默期 @ 充能

```
初始：2/2 满
被 @ → 消耗 1 次，回复
被 @ → 消耗 1 次，回复
被 @ → 0/2，忽略
每 15 分钟恢复 1 次（实时动态）
```

每个群独立计算。

### 9. 记忆系统

**用户记忆**（`data/memory/<qq_id>.md`）：

```
[0.9] 性格：话多，喜欢接梗
[0.8] 和我互动模式：经常互怼
[0.5] 最近在忙毕设
```

**群记忆**（`data/group_memory/<group_id>.md`）：

```
[0.9] 我在这个群：话多，爱接梗
[0.8] 核心人物：泷墨（话多）
```

- 结构化格式，按重要度分级（0.1-1.0）
- 注入 LLM 时截断 800 字，超 1500 字自动压缩
- 群聊和私聊记忆互通

**总结时机**：

| 触发 | 总结谁 |
|------|--------|
| session 结束 | 本轮被回复过的用户 + 群记忆 |
| 定时任务（6h） | 近 24 小时活跃用户 |
| `summarize` 指令 | 手动触发 |

### 10. 人格系统

`config/personality.yaml` 定义人格，`reload` 热重载。

- 20 条性格特征（二次元、社恐、废萌、拖延症、回避型...）
- 58 条风格规则（互联网语感、行为、情绪、聊天方式...）
- 7 条打字习惯（偶尔只回表情、突然消失、深夜碎碎念...）
- 5 种情绪示例（happy/sleepy/shy/emo/angry）
- 30+ 称呼库（熟人向/群友感/二次元/阴间互联网味）

### 11. 图片理解

- 普通图片 → URL 传入多模态 LLM
- 表情包（subType=1）→ 同样传入，但提示"不要较真"

### 12. LLM 决策

LLM 能看到：

| 内容 | 来源 |
|------|------|
| 系统提示词 | 人格 + 风格 + 情绪 + 打字习惯 + 禁词 |
| 用户记忆 | 数据库 + 长期记忆文件 |
| 群记忆 | 群记忆文件 |
| 聊天历史 | 最近 50 条（含 bot 发言，带名字标识） |
| 缓冲区消息 | 待决策的消息 |
| 疲惫提示 | 正常/敷衍/很累 |

LLM 返回 JSON：`{"replies": [{"message_id": 123, "text": "回复", "quote": true/false}]}`

## 快速开始

```bash
pip install -r requirements.txt
cp .env.example .env
# 编辑 .env 填入配置
python main.py
```

### NapCat 配置

1. 下载 [NapCatQQ](https://napneko.github.io/)，扫码登录
2. Web 面板（`http://127.0.0.1:6099`）配置：
   - HTTP 服务器：端口 `3000`
   - WebSocket 客户端：`ws://127.0.0.1:8765/onebot/ws`
3. 消息格式：**Array**
4. 配置 `NAPCAT_PATH` 可自动启动

## 终端指令

| 指令 | 说明 |
|------|------|
| `status` | 查看所有群状态 |
| `engage <群号> [0-100]` | 查看/设置参与度 |
| `say [群号] <消息>` | 以 bot 身份发群消息 |
| `summarize` | 总结所有用户+群记忆 |
| `summarize <qq_id>` | 总结指定用户 |
| `summarize group` | 总结群记忆 |
| `debug` | 切换实时状态面板 |
| `help` | 显示帮助 |

## 待开发

### 1. 心情系统

当前心情是每次回复时随机选取（happy/sleepy/shy/emo/angry），没有持续性。

计划：
- 心情有持续性，会随时间和事件变化
- 深夜更容易 emo，被夸会开心，被怼会生气
- 心情影响回复风格和出没频率
- 心情状态持久化，重启后保持

### 2. 记忆系统大改

当前记忆是纯文本文件 + LLM 总结，存在信息丢失和总结质量问题。

计划：
- 引入向量数据库（ChromaDB/Qdrant），支持语义检索
- 记忆分层：短期（聊天记录）→ 中期（近期总结）→ 长期（核心特征）
- 记忆衰减：不重要的记忆随时间淡化
- 主动回忆：根据当前话题检索相关记忆，而不是全量注入
- 关系图谱可视化
